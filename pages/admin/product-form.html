How this works

Setup fields become the buyer's setup form.

These fields are shown after checkout for self-serve products. For Nexus Guided Install, use these fields only for non-sensitive business information. Do not ask buyers to paste passwords here.

Use this for normal buyer information such as URLs, emails, report frequency, company notes, target audience, or business goals. In n8n, reference these as {{NEXUS_SETUP.field_name}}.
Use this only for future self-serve credential collection. For launch, most sensitive access should be handled through Nexus Guided Install instead of asking buyers to paste secrets. In n8n, reference these as {{NEXUS_SECRET.field_name}}.

Placeholder rules

{{NEXUS_SETUP.facebook_page_url}} Normal setup answer from the buyer form.
{{NEXUS_SECRET.facebook_access_token}} Sensitive credential field. Use carefully.
{{NEXUS_CUSTOMER.email}} Buyer profile or order data from Nexus.
{{NEXUS_SYSTEM.customer_automation_id}} Runtime IDs that Nexus fills automatically.

Map any placeholder used inside the uploaded n8n workflow to Nexus setup, secret, customer, or system fields. Nexus will replace these automatically during import.

Workflow rules

Before importing, check these basics.

  • The buyer result node should be named NEXUS_FINAL_OUTPUT.
  • Use {{NEXUS_SETUP.field}} for buyer setup answers.
  • Use {{NEXUS_SECRET.field}} only for sensitive credential placeholders.
  • Click Import any time. Nexus saves new products first, then imports automatically.

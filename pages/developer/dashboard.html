Workflow import guide

How Nexus imports and tests products

Use this before submitting a paid automation product. The importer adds the Nexus runtime parts, then your full workflow must pass a technical run.

Product flow

  1. Create or select a product draft.
  2. Paste the exported n8n workflow JSON. For Nexus testing, the workflow can start with a Manual Trigger.
  3. Add the buyer setup schema so Nexus knows what fields the customer must fill out.
  4. Save, import, and test. Nexus replaces/adds the webhook trigger, runtime context, and output callback.
  5. Add developer-owned provider keys in the credential vault, apply credentials, and run the check again.
  6. Submit for review only when the workflow is imported, credentials are bound, and the last run passed.

What Nexus adds during import

Nexus Webhook Trigger

Receives setup, buyer, order, schedule, and secure runtime data from Nexus.

Nexus Runtime Context

Makes buyer setup values available to Code nodes and expressions.

Nexus Submit Output

Sends the final report/output back to the buyer dashboard.

Name the final output node NEXUS_FINAL_OUTPUT when you want Nexus to attach the callback to a specific node.

Dynamic values you can use

Code node helpers 
NEXUS_CODE_SETUP("company_url")
NEXUS_CODE_SETUP("competitor_urls")
NEXUS_CODE_CUSTOMER("company")
NEXUS_CODE_CUSTOMER("name")
NEXUS_CODE_ORDER("id")
NEXUS_CODE_SYSTEM("customer_automation_id")
Expression placeholders 
{{NEXUS_SETUP.company_url}}
{{NEXUS_SETUP.competitor_urls}}
{{NEXUS_CUSTOMER.company}}
{{NEXUS_CUSTOMER.name}}
{{NEXUS_ORDER.id}}
{{NEXUS_SYSTEM.customer_automation_id}}

Recommended format is {{NEXUS_SETUP.field_name}}. Nexus also detects common variants such as {{Nexus_setup|field_name}}, {{NexusSetup.field_name}}, and {{NX_SETUP:field_name}}.

Approval rules

  • Standard paid products need workflow JSON, successful import, bound credentials, and a passed technical run.
  • Custom request products can be submitted without an automated runtime because they lead to an inquiry form.
  • If credentials change, run the full check again before the product can safely go live.
  • Nexus admin reviews product quality, buyer copy, setup clarity, and runtime readiness before publishing.
